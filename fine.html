Risk & Fraud Detection Dashboard — NARL
Designing a security monitoring interface for network analysts to detect and respond to threats faster at the National Atmospheric Research Laboratory.
3
Months duration
5+
User interviews
2
Prototype iterations

Problem

Security analysts at NARL were using Kerio Firewall's default interface — dense, data-heavy, and not optimised for rapid threat identification. Critical alerts were buried in noise, causing delayed responses to potential security incidents.

My role

  • Conducted 5 user interviews with network security analysts to understand daily workflows and pain points
  • Built user personas and mapped the current-state user journey to identify friction points
  • Created wireframes in Figma, iterated based on feedback, then delivered high-fidelity prototypes
  • Worked directly with the development team to hand off designs and verify implementation

Design decisions

  • Introduced a priority-based alert triage system — Critical / Warning / Info — so analysts scan top threats at a glance
  • Redesigned the data layout using progressive disclosure: summary view first, details on demand
  • Applied WCAG accessibility contrast ratios to all status indicators for low-light environments
  • Added a real-time activity timeline to give analysts chronological context for incidents

Outcome

The prototype was validated with 3 analysts who reported faster threat identification and reduced cognitive load compared to the original interface. The design was handed off to the development team for implementation.

E-learning Platform UI
End-to-end redesign of a digital learning platform to improve content discoverability, reduce drop-off, and make learning feel less overwhelming.
3
User flows mapped
12+
Screens designed
2

Problem

Learners struggled to find relevant courses, track their progress, and resume lessons mid-way. The navigation was inconsistent across devices, leading to high drop-off rates before course completion.

Process

  • Mapped 3 core user journeys: Course discovery → Enrolment → Learning & completion
  • Identified 6 key friction points through heuristic evaluation of the original interface
  • Sketched low-fidelity wireframes for each flow, then built interactive Figma prototypes
  • Ran 2 rounds of usability testing with 4 participants each and iterated on findings

Key design decisions

  • Introduced a persistent progress bar and "Continue learning" shortcut on the dashboard
  • Restructured navigation from 9 items to 4 primary categories using card sorting results
  • Designed a responsive layout system that works consistently across desktop, tablet, and mobile
  • Added micro-copy to guide first-time users through onboarding without a separate tutorial

Outcome

Usability test participants completed key tasks 40% faster in the redesigned prototype compared to the original. Course discovery time dropped significantly due to the improved search and filter system.

Cosmetics E-commerce App
Designing a mobile shopping experience that helps users discover, evaluate, and purchase cosmetics with confidence — reducing cart abandonment and decision fatigue.
5
Screens in checkout flow
20+

Core user journeys

Problem

Cosmetics shoppers online face two big problems: difficulty evaluating products without physical testing, and a checkout process with too many steps causing drop-off. The goal was to reduce friction across the entire purchase journey.

Process

  • Researched competitor apps (Nykaa, Sephora, Purplle) to identify UX patterns and gaps
  • Defined 3 primary user journeys: Browse & discover → Product evaluation → Checkout
  • Designed a complete component library: product cards, filters, reviews, cart, payment screens
  • Iterated on checkout flow to reduce steps from 6 to 3 using progressive disclosure

Key design decisions

  • Added shade finder and skin-type filter to help users narrow down products confidently
  • Designed a sticky "Add to bag" CTA that follows the user as they scroll product details
  • Introduced a visual order summary at checkout so users never feel uncertain about what they're buying
  • Built a consistent color system using the brand's warm pink palette across all states and components

Outcome

The streamlined 3-step checkout and contextual product filters resulted in a significantly cleaner experience validated in prototype testing. The visual design was consistently cited as feeling "premium and trustworthy" by test participants.

Risk & Fraud Detection Dashboard
A data-dense security monitoring dashboard designed for network analysts to identify threats, track incidents, and take action — built during my internship at NARL.
8+
Dashboard views
3

Challenge

Security monitoring tools are notoriously overwhelming. Analysts deal with hundreds of data points simultaneously and need to triage threats in seconds. The existing Kerio Firewall interface wasn't designed for this fast-paced, high-stakes workflow.

Solution highlights

  • Priority-tiered alert system: Critical (red) → Warning (amber) → Info (blue) for instant scanning
  • Real-time network activity timeline with drill-down capability for incident investigation
  • Clean data tables with sortable columns, inline actions, and bulk-select for analyst efficiency
  • Dark-mode-ready color palette with high contrast ratios for use in low-light server rooms

Tools used

  • Figma — wireframes, component library, high-fidelity prototype
  • FigJam — user journey mapping and stakeholder workshops
  • Miro — affinity mapping of user research findings
